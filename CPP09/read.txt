DIVIDE AND CONQUER

    En C++ (y en casi todos los lenguajes que usan recursión), una llamada recursiva “detiene” la ejecución de la función actual hasta que la llamada recursiva retorne.
Vamos a suponer que tenemos 20 números:

1️⃣ Primera iteración (nivel 0)

Entrada: 20 números.

Paso 1: Formamos pares (a,b) → 10 pares.
Paso 2: Identificamos el mayor de cada par → construimos sorted con 10 elementos (los mayores).
Paso 3: Llamada recursiva: sortContainer(sorted) con esos 10 elementos.
*En este momento, no se insertan los menores todavía, la inserción de los menores ocurre después de que la recursión haya ordenado la secuencia de mayores.*

2️⃣ Segunda iteración (nivel 1, recursión)

Entrada: sorted con 10 elementos.

Paso 1: Formamos pares → 5 pares.
Paso 2: Tomamos los mayores de cada par → construimos sorted con 5 elementos.
Paso 3: Llamada recursiva: sortContainer(sorted) con 5 elementos.
*Nuevamente, los menores de este nivel todavía no se insertan.*

(...)

4️⃣ Nivel base (último nivel)

Entrada: sorted con 1 o 2 elementos.

Caso base:
    1 elemento → ya está ordenado.
    2 elementos → se comparan y se ordenan directamente.

*Ahora empezamos a salir de la recursión, y aquí es donde se insertan los menores de cada par en la secuencia ordenada.*

5️⃣ Inserción de menores (al volver de la recursión)

Cada llamada recursiva retorna una secuencia base ordenada de mayores.

Los menores de esa llamada se insertan en posiciones correctas usando búsqueda binaria o algún método equivalente.

Este proceso ocurre en orden inverso a la recursión:

Nivel base → insert menores del último par
Nivel 2   → insert menores de pares de 5 mayores
Nivel 1   → insert menores de pares de 10 mayores
Nivel 0   → insert menores de pares de 20 originales


Así se asegura que cuando insertás un menor, la secuencia de mayores ya está ordenada, evitando comparaciones innecesarias.

TRY AND CATCH

El catch captura por tipo, no por mensaje.
try{}
catch (const NoOccurrence &e) {}    --> ⚠️ Esto captura solo excepciones del tipo NoOccurrence (o derivadas de ella).

si en mi clase defino mas tipos de errores(ex: class IndexError : public std::exception), el catch (const NoOccurrence &e) {}
no lo capturara

otra forma es tener varios catch


try {
    // código
}
catch (const NoOccurrence &e) {
    std::cerr << e.what();
}
catch (const IndexError &e) {
    std::cerr << e.what();
}
catch (const std::exception &e) {
    std::cerr << "Otro error: " << e.what();
}

⚠️El orden importa⚠️
C++ captura el primer catch compatible que encuentra.

esto sirve cuando necesitás hacer cosas distintas según el error.
